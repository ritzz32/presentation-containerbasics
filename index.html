Containers

The Basics

What is a Container?

A container image is a lightweight, stand-alone, executable package of a piece of software which includes everything needed to run it: code, runtime, system tools, system libraries, and settings.

  • Fills the same role as virtual machines

Why Use Containers?

  • Best to compare them to virtual machines

Virtual Machines vs Containers

Do One Thing and Do It Well.

  • Hardware virtualization vs OS virtualization
  • Added weight of OS and size of images
  • Direct access to hardware processing
  • Shared bins/libs between apps
  • VMs are fully isolated
  • Unix Philosophy - Do one thing and do it well
